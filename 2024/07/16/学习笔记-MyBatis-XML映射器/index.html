学习笔记 - MyBatis XML映射器

本文总结mapper.xml文件中主要标签的使用，及参数传递、结果映射等

标签

SQL 映射文件只有很少的几个顶级元素

  • cache – 该命名空间的缓存配置。
  • cache-ref – 引用其它命名空间的缓存配置。
  • resultMap – 描述如何从数据库结果集中加载对象，是最复杂也是最强大的元素。
  • sql – 可被其它语句引用的可重用语句块。
  • insert – 映射插入语句。
  • update – 映射更新语句。
  • delete – 映射删除语句。
  • select – 映射查询语句。

select

属性 描述
id 方法名，与命名空间的类名一同绑定Mapper接口的方法
parameterType 可选，方法参数类型
resultType 方法返回类型，如果是List，指定内部元素类型，如果是Map则指定为map
resultMap 结果映射，解决字段名不同、多对一、一对多，不能与resultType一同使用
flushCache 默认false，开启后该语句被调用则清空本地缓存和二级缓存
useCache select元素默认为true，开启后语句执行结果被二级缓存缓存
statementType 可选STATEMENT，PREPARED或CALLABLE

全部属性请参考：MyBatis中文网

insert｜update|delete

属性 描述
useGeneratedKeys 默认false（仅适用于 insert 和 update），使方法返回值由原来的影响行数变为主键id
keyProperty （仅适用于 insert 和 update），指定POJO的主键属性名
keyColumn （仅适用于 insert 和 update），指定数据表的主键字段名

参数

占位符

  • #{}，将其替换为？，防止sql注入

  • ${}，将其直接替换成参数

单个参数

都可以不使用@Param注解

类型 使用方式
基本类型 直接使用
POJO 直接使用，属性名和参数占位符名称一致
Map 直接使用，键名和参数占位符名称一致
Collection 底层封装为Map
键名：arg0 | collection
map.put(“arg0”, 集合)
map.put(“collection”, 集合)
List 底层封装为Map
map.put(“arg0”, 集合)
map.put(“collection”, 集合)
map.put(“list”, 集合)
Array 底层封装为Map
map.put(“arg0”, 数组)
map.put(“array”, 数组)

多个参数

  • 底层封装为Map集合

  • 第一个参数键名：arg0 | param1

  • 第二个参数键名：arg1 | param2

  • ……

  • 使用@Param()来替换arg的键名

最佳实践

  • 一个非数组或集合参数时不写@Param

  • 一个数组集合参数时写@Param

  • 引用类型属性与占位符一致

  • map类型键名与占位符一致

  • 多个参数时写@Param

resultType用于指定方法返回值的类型，不同返回值类型指定方式如下：

  • 基本类型：resultType=基本类型，默认写全限定名，可以指定别名

  • List类型：resultType=List中元素的类型

  • Map类型：resultType=map

resultMap

resultMap有许多的子元素，下面是一个较复杂的案例演示

resultMap常用子元素如下：

  • constructor - 用于在实例化类时，注入结果到构造方法中
    • idArg - 主键字段作为参数；标记出主键可以帮助提高整体性能
    • arg - 普通字段作为参数
  • id – 建立主键字段与 JavaBean 属性的映射；标记出主键可以帮助提高整体性能
  • result – 建立普通字段与 JavaBean 属性的映射
  • association – 关联：将一条记录的多个字段包装成一个对象作为单个属性
    • 嵌套结果映射 – 关联可以是 resultMap 元素，或是对其它结果映射的引用
  • collection – 集合：将多条记录的一部分字段收集为一个集合作为单个属性
    • 嵌套结果映射 – 集合可以是 resultMap 元素，或是对其它结果映射的引用
  • discriminator – 使用结果值来决定使用哪个 resultMap
    • case – 基于某些值的结果映射

constructor

constructor用于将查询结果的字段作为构造器参数来创建 JavaBean

注意：参数必须按照构造器顺序给出，从版本3.4.3开始可以在构造器使用@Param指定参数名并在arg标签指定name属性从而任意顺序

association

association用于将一条记录的多个字段包装成一个对象作为单个属性
