学习笔记 - Redis 入门

Redis诞生于2009年，全称是Remote Dictionary Server（远程词典服务器），是一个基于内存的键值型NoSQL数据库

特征

  • 键值型，valve支持多种不同数据结构，功能丰富
  • 单线程，每个命令具备原子性
  • 低延迟，速度快（基于内存、IO多路复用、良好的编码）
  • 支持数据持久化
  • 支持主从集群、分片集群
  • 支持多语言客户端

数据类型

Redis是一个key-value的数据库，key一般是String类型，valve的类型如下：

5种常用类型 描述
String 字符串
Hash 哈希
List 列表
Set 集合
Sorted Set/zset 有序集合
3种特殊类型 用途
GEO 地理坐标
BitMap 签到
HyperLogLog UV/PV

String

普通字符串

Hash

又称散列，是一个无序字典，类似 Java 中的 HashMap

String 类型存 json 字符串不便修改单个字段

Hash 结构将对象中的每个字段独立存储

List

与 Java 中的 LinkedList 类似，可以看做是一个双向链表结构

特征：

  • 有序
  • 元素可以重复
  • 插入和删除快
  • 查询速度一般

Set

与 Java 中的 HashSet 类似，可以看做是一个 value 为null的 HashMap

特征：

  • 无序
  • 元素不可重复
  • 查找快
  • 支持交集、并集、差集等功能

SortedSet

一个可排序的 set 集合，与 java 中的 TreeSet 类似，但底层数据结构差别很大。SortedSet 中的每个元素都带有一个 score 属性，基于 score 属性对元素排序，底层的实现是一个跳表（SkipList）加 hash 表

特征：

  • 可排序
  • 元素不重复
  • 查询时速度快

GEO

Geolocation的简写形式，代表地理坐标。Redis 在 3.2 版本中加入了对 GEO 的支持，允许存储地理坐标信息，根据经纬度来检索数据

BitMap

Redis 中利用 string 类型实现 BitMap，因此最大上限是512M，转换为bit则是2^32个bit

把每一位bit对应每一天，可以实现签到功能

HyperLogLog

HyperLogLog 是从 LogLog 算法派生的概率算法，用于确定非常大的集合的基数，而不需要存储其所有值。Redis 中的 HLL 是基于 string 结构实现的，单个 HLL 的内存永远小于16kb，测量误差小于0.81%

UV

Unique Vistor，独立访客量，指通过互联网访问、浏览网页的自然人。1天内同一个用户多次访问该网站，只记录一次

PV

全称 Page View，也叫页面访问量或点击量，用户每访问网站的一个页面，记录1次PV，用户多次打开页面，则记录多次PV。往往用来衡量网站的流量

Java客户端

Jedis

以 Redis 命令作为方法名称，学习成本低，简单实用，但是 Jedis 实例是线程不安全的，多线程环境下需要基于连接池

使用方式

  1. 导入依赖

  2. 方式一：直接使用

  3. 方式二：使用连接池

    JedisConnectionFactory

    JedisPoolTest

Lettuce

基于 Netty 实现的，支持同步、异步和响应式编程方式，线程安全。支持 Redis 的哨兵模式、集群模式和管道模式

Redisson

基于 Redis 实现的分布式、可伸缩的 Java 数据结构集合。包含了诸如 Map、Queue、Lock、Semaphore、AtomicLong 等强大功能

Spring Data Redis

SpringData 是 Spring 中数据操作的模块，包含对各种数据库的集成，其中对 Redis 的集成模块就叫做 SpringDataRedis

特征：

  • 封装了 Jedis 和 Lettuce
  • 提供了 RedisTemplate 统一 API
  • 支持 Redis 的发布订阅模型
  • 支持 Redis 哨兵和 Redis 集群
  • 支持基于 Lettuce 的响应式编程
  • 支持基于 JDK、JSON、字符串、Spring对象 的数据序列化及反序列化
  • 支持基于 Redis 的 JDKCollection 实现

使用方式：

  1. 导入 Maven 坐标

  2. 配置 Redis 数据源

  3. 通过 RedisTemplate 操作 Redis
