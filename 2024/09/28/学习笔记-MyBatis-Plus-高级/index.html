逻辑删除

逻辑删除是一种数据管理策略，通过一个表字段标记该记录是否被删除。MyBatis-Plus提供了便捷的逻辑删除支持，请参考逻辑删除支持 | MyBatis-Plus

工作原理

MyBatis-Plus 的逻辑删除功能会在执行数据库操作时自动处理逻辑删除字段。

  • 插入：逻辑删除字段的值不受限制。
  • 查找：自动添加条件，过滤掉标记为已删除的记录。
    • 例：select id,name,deleted from user where deleted=0
  • 更新：防止更新已删除的记录。
  • 删除：将删除操作转换为更新操作，标记记录为已删除。
    • 例：update user set deleted=1 where id = 1 and deleted=0

支持的数据类型

逻辑删除字段支持所有数据类型，但推荐使用 Integer、Boolean 或 LocalDateTime。如果使用 datetime 类型，可以配置逻辑未删除值为 null，已删除值可以使用函数如 now() 来获取当前时间。

使用方法

  1. 配置类中开启逻辑删除

    application.yml

  2. 实体类中使用@TableLogin注解标注逻辑删除字段

枚举映射

MP 在 mybatis 的 EnumOrdinalTypeHandler(基于枚举常量序号) 和 EnumTypeHandler(基于枚举常量名) 之外，提供了更加灵活的枚举处理器 MybatisEnumTypeHandler(基于枚举常量属性)，只需要对枚举进行声明，即可实现枚举的自动映射，未进行声明的枚举则根据 mybatis的defaultEnumTypeHandler 的默认值EnumTypeHandler 来进行映射，详情请参考自动映射枚举 | MyBatis-Plus

使用

  1. 方式一：注解标记

    枚举属性使用 @EnumValue 注解，指定枚举值在数据库中存储的实际值。支持枚举类中的任意字段，如序号或编码。

  2. 方式二：实现接口

    实现 IEnum 接口，实现 getValue 方法，指定枚举值在数据库中存储的实际值。支持枚举类中的任意字段，如序号或编码。
