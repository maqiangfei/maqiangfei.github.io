学习笔记 - MyBatis XML映射器

本文总结mapper.xml文件中主要标签的使用，及参数传递、结果映射等

标签

SQL 映射文件只有很少的几个顶级元素

  • cache – 该命名空间的缓存配置。
  • cache-ref – 引用其它命名空间的缓存配置。
  • resultMap – 描述如何从数据库结果集中加载对象，是最复杂也是最强大的元素。
  • sql – 可被其它语句引用的可重用语句块。
  • insert – 映射插入语句。
  • update – 映射更新语句。
  • delete – 映射删除语句。
  • select – 映射查询语句。

select

属性 描述
id 方法名，与命名空间的类名一同绑定Mapper接口的方法
parameterType 可选，方法参数类型
resultType 方法返回类型，如果是数组或集合，只用看内部元素的类型
resultMap 结果映射，解决字段名不同、多对一、一对多，不能与resultType一同使用
flushCache 默认false，开启后该语句被调用则清空本地缓存和二级缓存
useCache select元素默认为true，开启后语句执行结果被二级缓存缓存
statementType 可选 STATEMENT，PREPARED 或 CALLABLE

全部属性请参考：MyBatis中文网

insert｜update|delete

属性 描述
useGeneratedKeys 默认false（仅适用于 insert 和 update），使方法返回值由原来的影响行数变为主键id
keyProperty （仅适用于 insert 和 update），指定POJO的主键属性名
keyColumn （仅适用于 insert 和 update），指定数据表的主键字段名

sql

这个元素可以用来定义可重用的 SQL 代码片段，以便在其它语句中使用。

参数

占位符

  • #{}，将其替换为？，防止sql注入

  • ${}，将其直接替换成参数

单个参数

都可以不使用@Param注解

类型 使用方式
基本类型 直接使用
POJO 直接使用，属性名和参数占位符名称一致
Map 直接使用，键名和参数占位符名称一致
Collection 底层封装为Map
键名：arg0 | collection
map.put(“arg0”, 集合)
map.put(“collection”, 集合)
List 底层封装为Map
map.put(“arg0”, 集合)
map.put(“collection”, 集合)
map.put(“list”, 集合)
Array 底层封装为Map
map.put(“arg0”, 数组)
map.put(“array”, 数组)

多个参数

  • 底层封装为Map集合

  • 第一个参数键名：arg0 | param1

  • 第二个参数键名：arg1 | param2

  • ……

  • 使用@Param()来替换arg的键名

最佳实践

  • 一个非数组或集合参数时不写@Param

  • 一个数组集合参数时写@Param

  • 引用类型属性与占位符一致

  • map类型键名与占位符一致

  • 多个参数时写@Param
